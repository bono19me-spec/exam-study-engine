# 시험용 학습 엔진 웹앱 기획서

## 0. 프로젝트 개요

### 프로젝트명
**AI Study Engine**

### 첫 번째 콘텐츠팩
**秘書検定2級 Content Pack**

### 목표
특정 자격증 하나에 종속된 앱이 아니라, 여러 시험 콘텐츠를 교체해서 사용할 수 있는 **범용 시험 학습 엔진**을 만든다.  
첫 번째 적용 대상은 일본 **秘書検定2級**이며, 이후 콘텐츠팩만 추가하면 다른 시험으로 확장할 수 있게 설계한다.

```text
AI Study Engine
 ├─ content/hisho2        秘書検定2級
 ├─ content/hisho3        秘書検定3級
 ├─ content/boki3         簿記3級
 ├─ content/boki2         簿記2級
 └─ content/jlpt_n3       JLPT N3
```

## 1. 핵심 설계 원칙

1. 앱 코어는 시험 공통 기능만 담당한다.
2. 시험별 이론, 문제, 태그, 모의고사 설정은 콘텐츠팩에서 관리한다.
3. 원본 콘텐츠는 기본적으로 일본어로 작성한다.
4. 번역은 원본 데이터를 바꾸지 않고, 현재 화면을 임시 번역하는 보조 기능으로 둔다.
5. 문제와 해설은 직접 제작한 콘텐츠를 사용한다.
6. 외부 무료 자료는 링크와 참고용으로만 사용하고, 문제·해설을 그대로 복사하지 않는다.
7. 문제 형식, 선택지 수, 합격 기준, 제한 시간은 시험별 설정으로 관리한다.

## 5. 콘텐츠 언어 정책

### 원본 콘텐츠
모든 원본 콘텐츠는 일본어로 작성한다.

```text
이론: 일본어
문제: 일본어
선택지: 일본어
해설: 일본어
태그: 일본어
UI 기본 문구: 일본어
```

### 번역 기능
사용자가 `[翻訳]` 버튼을 누르면 현재 페이지의 일본어 텍스트를 선택한 언어로 임시 번역해서 보여준다.

```text
번역 대상:
- 현재 Lesson 본문
- 문제 본문
- 선택지
- 정답 해설
- 오답 해설
```

번역 결과는 학습 보조용이며, 정답 판단의 기준은 항상 일본어 원문이다.

## 6. 무료 번역 API 후보

무료 API는 제한이 바뀔 수 있으므로, 구현 시점에 다시 확인한다.

### 후보 1: LibreTranslate
- 오픈소스 번역 API
- 무료 공개 서버가 있으나 사용량 제한과 안정성 문제가 있을 수 있음
- 가능하면 직접 서버를 띄우는 방식도 고려

### 후보 2: MyMemory Translation API
- 간단한 무료 번역 API
- 무료 사용량 제한 있음
- 품질은 문장에 따라 차이가 있음

### 후보 3: 브라우저 내장 번역 유도
- API 실패 시 사용자가 브라우저 번역 기능을 쓰도록 안내
- 가장 단순한 fallback

### 번역 처리 방식

```text
[翻訳] 클릭
 ↓
현재 화면의 텍스트 추출
 ↓
API 요청
 ↓
번역 결과 표시
 ↓
[原文に戻す] 클릭 시 일본어 원문 복귀
```

번역 결과는 캐시한다.

```text
translation_cache:
- sourceTextHash
- sourceLanguage
- targetLanguage
- translatedText
- createdAt
```

## 23. 가장 중요한 방향

이 앱은 **비서검정 전용 앱**이 아니라 **시험 공부를 위한 범용 엔진**이다.

비서검정 2급은 첫 번째 콘텐츠팩일 뿐이다.

```text
앱 = 학습 엔진
콘텐츠 = 시험별 교재와 문제은행
```

이 구조로 만들면 앞으로 부기, JLPT, FP, 서비스접우검정 등도 같은 앱에서 확장할 수 있다.
